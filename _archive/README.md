# _archive

This directory holds files that have been retired from the active project.
Each subfolder has its own README explaining what it contained, why it was
archived, and when it may be safe to delete.

Nothing in `_archive/` is loaded by the framework, referenced by CI, or
required at runtime. It exists purely as a historical reference.

## Contents

| Folder | What was archived |
|--------|-------------------|
| `poc_renderer.py` | Standalone proof-of-concept that preceded the `askee_ds` package. |
| `design-ascii/` | The original U+241F delimiter format files (components.txt, format spec, tokens, manifest, prop shapes). |
| `tools/` | Retired CLI tools and migration scripts that operated on the old format. |
